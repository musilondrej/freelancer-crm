# Code of Conduct

## Purpose

Freelancer CRM is an open source project for practical collaboration. The goal of this code of conduct is to keep the project respectful, constructive, and safe for contributors, maintainers, and users.

## Expected Behavior

When participating in project spaces, contributors are expected to:

- communicate respectfully and assume good intent until there is evidence otherwise,
- give technical feedback about ideas and implementation, not about people,
- be open to correction and willing to adjust when new information appears,
- keep discussions relevant, practical, and readable,
- respect differences in experience level, language, background, and perspective.

## Unacceptable Behavior

The following behavior is not acceptable in project spaces:

- harassment, intimidation, or personal attacks,
- discriminatory or exclusionary language,
- trolling, bad-faith argumentation, or repeated derailing of discussions,
- publishing private information without permission,
- sexualized language or unwanted advances,
- sustained hostile behavior after a request to stop.

## Scope

This code of conduct applies to:

- issues,
- pull requests,
- discussions,
- commit messages,
- code review interactions,
- any other official project communication channels.

## Enforcement

Maintainers may take any action they consider appropriate to protect the project and its participants, including:

- requesting a behavior change,
- editing or removing inappropriate content,
- locking threads,
- rejecting contributions,
- temporarily or permanently restricting participation.

Enforcement decisions will be based on impact to the project and its participants, not only on intent.

## Reporting

If you experience or witness behavior that violates this code of conduct, report it privately to the maintainers through the contact channel listed in [SECURITY.md](SECURITY.md) or the primary repository contact method.

When reporting, include as much relevant context as possible:

- what happened,
- where it happened,
- who was involved,
- links or screenshots if available.

Reports will be handled as discreetly as practical.

## Project Standard

The baseline expectation is simple: be direct, be respectful, and keep disagreements technical.
